Microphone Sensitivity GUI — UPV Audio Analyzer Automation

Bachelor Thesis Project Overview

1. What This Project Is

This is a Python desktop application that provides a graphical interface to control and automate a Rohde & Schwarz UPV Audio Analyzer — a precision laboratory instrument used for acoustic measurements. Instead of manually operating the UPV's complex front panel for every measurement, this software automates the entire workflow: configuring the instrument, running frequency sweeps, capturing data in real-time, and exporting results in a standardized format.

2. The Problem It Solves

Performing microphone sensitivity measurements (and related tests like OSPL90, FOG, etc.) on the UPV requires:

  • Manually entering dozens of SCPI instrument commands via the front panel
  • Waiting for sweeps to complete with no visual feedback on a PC
  • Manually exporting data after each measurement
  • Repeating the entire process for every measurement preset

This GUI eliminates this manual process by letting the user load a preset, click a button, and receive a plotted result plus an exportable data file — all from a single application window.

5. Data Flow — End-to-End Workflow

Step 1: Connect

The application scans VISA resources (LAN/USB), discovers the UPV instrument, and saves the connection address to config.json for future sessions.

Step 2: Configure

The user selects a JSON preset file (e.g., mic_sensitivity.json). The preset defines all instrument parameters organised into four sections: Generator Config, Generator Function, Analyzer Config, and Analyzer Function.

Step 3: Apply Settings

The application translates each JSON key into a SCPI command and sends it to the UPV. For example:

  • "Start": "100 Hz" → SOUR:SWE:FREQ:STAR 100 Hz
  • "Function Generator": "SIN" → SOUR:FUNC SIN
  • "Output Type (Unbal/Bal)": "UNB" → OUTP:TYPE UNB

Step 4: Sweep

The user clicks "Start Sweep". A background thread polls trace data from the UPV in real time, pushing it through a thread-safe queue to the GUI for live plotting. The user sees the frequency response curve building point-by-point.

Step 5: Export

Results are saved as .hxml files (an XML format containing frequency + level arrays with metadata) and displayed as a Matplotlib plot with a logarithmic X-axis.

7. HXML Output Format

Results are exported as .hxml (XML-based), a format compatible with downstream acoustic analysis tools:

<?xml version="1.0" encoding="utf-8"?>
<hxml>
  <head>
    <Document>
      <DataType>hiCurve</DataType>
    </Document>
  </head>
  <data>
    <dataset WorkingTitle="workingTitle">
      <v-curvedata>
        <curvedata
          CurveDataName="mic_sensitivity"
          MeasurementDate="19-Nov-2025 09:41:44"
          TestEquipmentNr="UPV_Audio_Analyzer"
          Tester="PythonApp">
          <curve name="f" unit="Hz">
            [100.000 105.441 ... 20000.000]
          </curve>
          <curve name="level" unit="dBr">
            [114.354 114.319 ... 59.088]
          </curve>
        </curvedata>
      </v-curvedata>
    </dataset>
  </data>
</hxml>

Each file contains two parallel data arrays: frequency points (Hz) and corresponding measured levels (in the configured unit — dBV, dBr, dB SPL, etc.).

8. GUI Features

Main Window (4-Panel Layout)

  • Generator Config panel — Instrument selection, output type, impedance, voltage range, reference values
  • Generator Function panel — Waveform type, sweep control, frequency range, spacing, voltage
  • Analyzer Config panel — Input channel, coupling, bandwidth, impedance, trigger settings
  • Analyzer Function panel — Measurement function (RMS, THD, FFT, etc.), filters, settling, monitors

Control Buttons

  • Connect to UPV — Automatic instrument discovery on LAN/USB
  • Apply Settings — Send all GUI parameters to instrument
  • Start Sweep — Begin measurement with live plot display
  • Stop Continuous — Halt continuous sweep mode
  • Save Preset / Load Preset — Manage measurement configurations
  • Snapshot Settings — Read back current instrument state

Multi-Preset Sequencing

  • Inline checkbox list of available measurement presets
  • Drag/reorder with Up/Down buttons
  • Remove/Restore items from execution order
  • "Apply Selected" runs all chosen presets sequentially
  • Each preset gets its own live plot window
  • Combined HXML export after sequence completes

9. Notable Engineering Decisions

AspectApproach
ConcurrencyBackground acquisition thread + thread-safe VISA mutex lock + queue-based producer/consumer pattern — keeps the GUI responsive during live sweeps
Multi-Preset SequencingUsers can tick multiple presets, reorder them, and run all sequentially — each preset gets its own live plot window
Unit HandlingAutomatic normalization of units across presets (dBV, dBu, dBm, dB SPL, μV, %, etc.) with on-the-fly conversion in the GUI
PackagingPyInstaller build script produces a standalone .exe with all presets bundled — no Python installation required on the measurement PC
RobustnessGlobal crash logger, timeout-guarded SCPI queries, graceful fallbacks when instrument disconnects mid-measurement
Value NormalizationSnapshot and preset values are automatically cleaned (e.g. "PCT" → "%", "DBV" → "dBV", "uV" → "μV") for consistent display
