Presentation Notes — Microphone Sensitivity GUI

1. Project Introduction

  • Built a Python desktop application to automate acoustic measurements on an R&S UPV Audio Analyzer
  • Replaces the manual, repetitive process of configuring the instrument and collecting data
  • Provides a single GUI window to configure, measure, visualise, and export — all in one place
  • Packaged as a standalone Windows .exe — no Python installation needed on the lab PC

2. Problem Statement

  • The UPV Audio Analyzer requires dozens of manual SCPI commands to set up each measurement
  • Each test type (sensitivity, OSPL90, FOG, etc.) needs a different configuration — error-prone to enter by hand
  • No built-in way to sequence multiple measurements back-to-back automatically
  • No live visual feedback on a PC while a sweep is running
  • Manual data export is time-consuming and inconsistent

3. Solution Overview

  • Store each measurement configuration as a JSON preset file
  • GUI reads the preset → translates to SCPI commands → sends to UPV via LAN or USB
  • One click to apply settings, one click to start the sweep
  • Live plot updates in real-time as the instrument sweeps
  • Automatic export to .hxml format (compatible with downstream analysis tools)
  • Can queue multiple presets and run them in sequence without manual intervention

4. System Architecture

  • GUI Layer (Tkinter) — what the user sees and interacts with
    • 4-panel layout: Generator Config, Generator Function, Analyzer Config, Analyzer Function
    • Each panel is independently scrollable with labelled controls
  • Display Mapping Layer — translates between machine codes and human-readable labels
    • Example: ANLG → "Analog", UNB → "Unbalanced", ASW → "Auto Sweep"
  • Instrument Communication Layer (PyVISA) — talks to the physical UPV
    • Sends SCPI commands over LAN (TCP/IP) or USB
    • Reads back trace data (frequency + level arrays)
  • Output Layer — saves results as .hxml files + generates Matplotlib plots

5. How It Works (Step by Step)

  • Step 1 — Connect: App auto-discovers the UPV on the network (scans all VISA resources)
  • Step 2 — Load Preset: User selects a JSON file (e.g. mic_sensitivity.json)
  • Step 3 — Apply Settings: Each JSON key is translated to a SCPI command and sent to the UPV
    • "Start": "100 Hz" becomes SOUR:SWE:FREQ:STAR 100 Hz
    • "Function Generator": "SIN" becomes SOUR:FUNC SIN
  • Step 4 — Start Sweep: Background thread fetches live data → plots update in real-time
  • Step 5 — Export: Data saved as .hxml file containing frequency points + measured levels

6. Key Features

  • Automatic instrument discovery — no need to manually type IP addresses
  • Preset-based configuration — each test type is a simple JSON file, easy to create and share
  • Live sweep visualisation — frequency response curve draws in real-time during measurement
  • Multi-preset sequencing — tick multiple presets, reorder them, run all with one click
  • Unit conversion in the GUI — switch between V, mV, μV, dBV, dBu, dBm on the fly
  • Snapshot readback — capture the current instrument state as a JSON file for reference
  • Standalone .exe — packaged with PyInstaller, runs on any Windows PC without setup

7. Measurement Presets Available

  • mic_sensitivity.json — Standard microphone sensitivity (100 Hz to 20 kHz, 101 log-spaced points)
  • OSPL90.json — Output Sound Pressure Level at 90 dB (dB SPL units)
  • COP_Sensitivity.json — Sensitivity test with 10 Ω output impedance
  • FOGm20.json — Free-field Open-circuit Gain at −20 dB
  • FOG_Li50.json — FOG at Li50 condition
  • sweep12k.json — Single-frequency test at 12 kHz (calibration/verification)
  • 1k@70k.json — 1 kHz tone with 70 kHz bandwidth

8. Output Format (.hxml)

  • XML-based file format used by acoustic analysis tools
  • Contains two data arrays:
    • Frequency points in Hz (e.g. 100, 105, 111, ... 20000)
    • Corresponding level values in the configured unit (dBV, dBr, dB SPL, etc.)
  • Includes metadata: measurement date, equipment ID, tester name, preset name
  • Multiple measurements can be combined into a single .hxml after a sequence

9. Technical Challenges & Solutions

  • Challenge: GUI freezes while waiting for instrument responses
    • Solution: Background acquisition thread + thread-safe queue (producer-consumer pattern)
  • Challenge: Concurrent VISA access causes communication errors
    • Solution: Mutex lock (threading.Lock) around all instrument queries
  • Challenge: Inconsistent unit formatting across presets and snapshots
    • Solution: Automatic normalization layer (e.g. "PCT" → "%", "DBV" → "dBV", "uV" → "μV")
  • Challenge: Application must work on lab PCs without Python installed
    • Solution: PyInstaller bundles everything into a standalone .exe
  • Challenge: Instrument may disconnect mid-measurement
    • Solution: Timeout-guarded queries, graceful error handling, crash logging to file

10. Technologies Used

  • Python 3 — main programming language
  • Tkinter — GUI framework (built into Python, no extra install)
  • PyVISA — VISA library for SCPI instrument communication over LAN/USB
  • Matplotlib — real-time plotting of frequency response curves
  • NumPy — numerical data array handling
  • PyInstaller — packaging into standalone Windows executable
  • JSON — preset configuration file format
  • SCPI — Standard Commands for Programmable Instruments (industry protocol)

11. Project Structure

  • mic-sensitivity-gui/src/ — all application source code
    • main.py — entry point
    • gui/window.py — main GUI window (~4000 lines)
    • gui/display_map.py — SCPI code ↔ label mappings
    • upv/upv_auto_config.py — instrument driver & data export
    • upv/upv_readback.py — instrument state readback
    • utils/paths.py — path resolution (dev vs. packaged)
  • *.json files — measurement preset configurations
  • Results/ — exported .hxml measurement files (output artifacts)
  • build/ — PyInstaller build output
  • main script/ — original command-line version (before GUI was developed)

12. Summary / Conclusion

  • Developed a complete automation tool for the R&S UPV Audio Analyzer
  • Reduces measurement setup time from minutes of manual entry to a single click
  • Supports multiple standardised test types through a preset system
  • Provides real-time visual feedback during measurements
  • Exports results in a standardised .hxml format for downstream analysis
  • Handles concurrency, error recovery, and unit normalization robustly
  • Packaged as a standalone executable for easy deployment in the lab
